Unit 13

Menganalisis User Interface

Kode Unit: J.62SAD00.013.1

Tujuan Pembelajaran

Setelah menyelesaikan unit ini, peserta mampu:

  • Menganalisis user experience (UX)
  • Mendesain user interface (UI)
  • Membuat wireframe dan prototype
  • Melakukan usability testing
  • Membuat UI specifications

User Experience (UX) Design

Definisi UX Design

UX Design adalah proses merancang produk yang memberikan pengalaman yang bermakna dan relevan kepada pengguna melalui interaksi dengan produk tersebut.

Prinsip UX Design

User-Centered Design

  • Penelitian pengguna
  • Persona pengguna
  • Pemetaan perjalanan pengguna
  • Pengujian pengguna

Prinsip Usabilitas

  • Kemudahan belajar
  • Efisiensi
  • Kemudahan mengingat
  • Pencegahan kesalahan

Proses UX Design

Penelitian
  • Penelitian pengguna
  • Penelitian pasar
  • Analisis kompetitif
  • Pengumpulan kebutuhan
Desain
  • Arsitektur informasi
  • Wireframing
  • Prototyping
  • Desain visual
Pengujian
  • Pengujian kegunaan
  • Umpan balik pengguna
  • Iterasi
  • Validasi
Implementasi
  • Pengembangan
  • Jaminan kualitas
  • Peluncuran
  • Pemantauan

User Research

Metode Penelitian

  • Wawancara: Wawancara pengguna satu-satu
  • Survei: Survei online dan offline
  • Focus Group: Diskusi kelompok
  • Observasi: Observasi perilaku pengguna

User Personas

  • Persona Utama: Pengguna target utama
  • Persona Sekunder: Pengguna target sekunder
  • Persona Negatif: Pengguna yang dihindari
  • Detail Persona: Demografi, tujuan, pain points

User Journey Mapping

  • Kesadaran: Bagaimana pengguna menemukan produk
  • Pertimbangan: Bagaimana pengguna mengevaluasi produk
  • Pembelian: Bagaimana pengguna memperoleh produk
  • Penggunaan: Bagaimana pengguna berinteraksi dengan produk
  • Retensi: Bagaimana pengguna terus menggunakan produk

User Interface (UI) Design

Definisi UI Design

UI Design adalah proses merancang interface yang memungkinkan pengguna berinteraksi dengan produk secara efektif dan efisien.

Prinsip UI Design

Hierarki Visual
  • Ukuran dan skala
  • Kontras warna
  • Tipografi
  • Spasi
Konsistensi
  • Pola desain
  • Skema warna
  • Tipografi
  • Struktur layout

Prinsip UI Design (Lanjutan)

Aksesibilitas
  • Kontras warna
  • Navigasi keyboard
  • Dukungan screen reader
  • Teks alternatif
Responsivitas
  • Desain mobile-first
  • Layout fleksibel
  • Konten adaptif
  • Interface ramah sentuh

UI Design Elements

Elemen Visual

  • Tipografi: Font, ukuran, ketebalan
  • Warna: Palet warna, kontras
  • Gambar: Foto, ikon, ilustrasi
  • Bentuk: Tombol, kartu, kontainer

Elemen Interaktif

  • Tombol: Primer, sekunder, tersier
  • Form: Field input, label, validasi
  • Navigasi: Menu, breadcrumb, paginasi
  • Umpan Balik: Status loading, pesan error

Wireframing

Low-Fidelity
  • Layout dasar
  • Struktur konten
  • Alur navigasi
  • Iterasi cepat
Mid-Fidelity
  • Layout detail
  • Hierarki konten
  • Elemen interaktif
  • Alur pengguna
High-Fidelity
  • Desain pixel-perfect
  • Konten final
  • Prototype interaktif
  • Siap pengembangan

Prototyping

Jenis Prototype

  • Prototype Statis: Mockup non-interaktif
  • Prototype Interaktif: Interface yang dapat diklik
  • Prototype Animasi: Gerakan dan transisi
  • Prototype Fungsional: Fitur yang berfungsi

Prototyping Tools

  • Tools Desain: Figma, Sketch, Adobe XD
  • Tools Prototyping: InVision, Marvel, Principle
  • Tools Kode: HTML/CSS, React, Vue
  • Tools Pengujian: UserTesting, Maze, Lookback

Usability Testing

Pengujian Terpandu
  • Fasilitator hadir
  • Umpan balik real-time
  • Wawasan detail
  • Biaya lebih tinggi
Pengujian Tidak Terpandu
  • Mandiri
  • Pengujian jarak jauh
  • Skalabel
  • Biaya lebih rendah

Metode Pengujian

  • Pengujian Berbasis Tugas: Penyelesaian tugas spesifik
  • Think-Aloud Protocol: Umpan balik verbal
  • Eye Tracking: Analisis perhatian visual
  • A/B Testing: Pengujian komparatif

Metrik Pengujian

  • Tingkat Keberhasilan Tugas: Persentase tugas yang diselesaikan
  • Waktu Tugas: Waktu untuk menyelesaikan tugas
  • Tingkat Error: Jumlah kesalahan yang dibuat
  • Kepuasan Pengguna: Skor umpan balik pengguna

UI Specifications

Spesifikasi Visual
  • Kode warna
  • Tipografi
  • Spasi
  • Dimensi
Spesifikasi Interaktif
  • Status hover
  • Status klik
  • Transisi
  • Animasi
Spesifikasi Responsif
  • Breakpoint

  • Variasi layout
  • Skala konten

Latihan Praktis

Latihan 1: Penelitian UX

Tugas: Lakukan penelitian UX untuk sistem perpustakaan

Langkah:
  1. Identifikasi pengguna target
  2. Buat persona pengguna
  3. Peta perjalanan pengguna
  4. Identifikasi pain points
  5. Buat laporan penelitian

Latihan 2: Desain UI

Tugas: Desain UI untuk sistem perpustakaan

Langkah:
  1. Buat wireframe
  2. Desain interface visual
  3. Buat prototype interaktif
  4. Uji dengan pengguna
  5. Iterasi berdasarkan umpan balik

Latihan 3: Pengujian Kegunaan

Tugas: Lakukan pengujian kegunaan

Langkah:
  1. Buat rencana pengujian
  2. Rekrut peserta pengujian
  3. Lakukan pengujian kegunaan
  4. Analisis hasil
  5. Buat laporan pengujian

Latihan 4: Spesifikasi UI

Tugas: Buat spesifikasi UI lengkap

Langkah:
  1. Dokumentasikan spesifikasi visual
  2. Dokumentasikan spesifikasi interaktif
  3. Dokumentasikan spesifikasi responsif
  4. Buat panduan gaya
  5. Validasi dengan tim pengembangan

Proyek Integrasi: User Interface

Analisis UI untuk Sistem Perpustakaan Digital:

1. UX Research

Persona Pengguna

  • Persona Utama - Mahasiswa: Pengguna perpustakaan aktif, melek teknologi, mobile-first
  • Persona Sekunder - Pustakawan: Staf perpustakaan, pengguna desktop, fokus efisiensi
  • Persona Sekunder - Dosen: Fokus penelitian, pengguna desktop, fitur lanjutan

User Journey

Perjalanan Peminjaman Mahasiswa

  • Penemuan: Mencari buku
  • Evaluasi: Periksa detail dan ketersediaan buku
  • Aksi: Pinjam atau reservasi buku
  • Manajemen: Lacak riwayat peminjaman
  • Pengembalian: Kembalikan buku tepat waktu

Pain Points

  • Masalah Pencarian: Sulit menemukan buku
  • Masalah Ketersediaan: Ketersediaan buku tidak jelas
  • Pengalaman Mobile: Interface mobile yang buruk
  • Masalah Notifikasi: Tidak ada notifikasi real-time

2. UI Design

Sistem Desain

  • Palet Warna: Biru primer, hijau sekunder, abu-abu netral
  • Tipografi: Font family Inter, ukuran dasar 16px
  • Spasi: Sistem grid 8px
  • Komponen: Gaya tombol, form, dan kartu yang konsisten

Layout Design

  • Header: Logo, navigasi, menu pengguna
  • Sidebar: Akses cepat, filter, kategori
  • Konten Utama: Daftar buku, hasil pencarian
  • Footer: Link, informasi kontak

Responsive Design

  • Mobile (320px-768px): Satu kolom, ramah sentuh
  • Tablet (768px-1024px): Dua kolom, navigasi hibrida
  • Desktop (1024px+): Multi-kolom, navigasi lengkap

3. Wireframes

Halaman Utama

  • Beranda: Pencarian, buku unggulan, akses cepat
  • Hasil Pencarian: Daftar buku, filter, paginasi
  • Detail Buku: Informasi buku, ketersediaan, aksi
  • Dashboard Pengguna: Riwayat peminjaman, reservasi, denda

Prototypes

Prototype Interaktif

  • Low-Fidelity: Layout dan navigasi dasar
  • Mid-Fidelity: Interaksi detail
  • High-Fidelity: Desain final dengan animasi

4. Usability Testing

Skenario Pengujian

  • Skenario 1: Cari dan pinjam buku
  • Skenario 2: Reservasi buku yang tidak tersedia
  • Skenario 3: Periksa riwayat peminjaman
  • Skenario 4: Bayar denda online

Test Results

Temuan Utama

  • Tingkat Keberhasilan Tugas: 85%
  • Waktu Rata-rata: 2.5 menit per tugas
  • Tingkat Error: 15%
  • Kepuasan Pengguna: 4.2/5

Issues Identified

  • Interface Pencarian: Filter pencarian membingungkan
  • Navigasi Mobile: Sulit mengakses menu
  • Detail Buku: Informasi kunci hilang
  • Proses Peminjaman: Terlalu banyak langkah

5. UI Specifications

Spesifikasi Visual

  • Warna: Primer #3498db, Sekunder #27ae60, Error #e74c3c
  • Tipografi: Inter Regular 16px, Bold 18px
  • Spasi: 8px, 16px, 24px, 32px
  • Komponen: Tinggi tombol 44px, Tinggi input 40px

Spesifikasi Interaktif

  • Status Hover: Transisi 0.2s, perubahan warna
  • Status Klik: Transisi 0.1s, efek skala
  • Status Loading: Skeleton screen, progress bar
  • Status Error: Border merah, pesan error

Spesifikasi Responsif

  • Breakpoint: 320px, 768px, 1024px, 1440px
  • Sistem Grid: Grid 12 kolom, gutter 20px
  • Skala Tipografi: 14px mobile, 16px tablet, 18px desktop
  • Skala Komponen: Tombol responsif, layout fleksibel

Style Guide

Library Komponen

  • Tombol: Variant primer, sekunder, outline
  • Form: Field input, label, validasi
  • Kartu: Kartu buku, kartu pengguna, kartu info
  • Navigasi: Header, sidebar, breadcrumb

Accessibility

  • Kontras Warna: Kepatuhan WCAG AA
  • Navigasi Keyboard: Urutan tab, status fokus
  • Screen Reader: Label ARIA, teks alt
  • Target Sentuh: Area sentuh minimum 44px
