Unit 13

Menganalisis User Interface

Kode Unit: J.62SAD00.013.1

Tujuan Pembelajaran

Setelah menyelesaikan unit ini, peserta mampu:

  • Menganalisis user experience (UX)

User Experience (UX) Design

Definisi UX Design

UX Design adalah proses merancang produk yang memberikan pengalaman yang bermakna dan relevan kepada pengguna melalui interaksi dengan produk tersebut.

UX Design Principles

User-Centered Design

  • User research
  • User personas
  • User journey mapping
  • User testing

Usability Principles

  • Learnability
  • Efficiency
  • Memorability
  • Error prevention

UX Design Process

Research
  • User research
  • Market research
  • Competitive analysis
  • Requirements gathering
Design
  • Information architecture
  • Wireframing
  • Prototyping
  • Visual design
Testing
  • Usability testing
  • User feedback
  • Iteration
  • Validation
Implementation
  • Development
  • Quality assurance
  • Launch
  • Monitoring

User Research

Research Methods

  • Interviews: One-on-one user interviews
  • Surveys: Online and offline surveys
  • Focus Groups: Group discussions
  • Observations: User behavior observation

User Personas

  • Primary Personas: Main target users
  • Secondary Personas: Secondary target users
  • Negative Personas: Users to avoid
  • Persona Details: Demographics, goals, pain points

User Journey Mapping

  • Awareness: How users discover the product
  • Consideration: How users evaluate the product
  • Purchase: How users acquire the product
  • Usage: How users interact with the product
  • Retention: How users continue using the product

User Interface (UI) Design

Definisi UI Design

UI Design adalah proses merancang interface yang memungkinkan pengguna berinteraksi dengan produk secara efektif dan efisien.

UI Design Principles

Visual Hierarchy
  • Size and scale
  • Color contrast
  • Typography
  • Spacing
Consistency
  • Design patterns
  • Color schemes
  • Typography
  • Layout structure

UI Design Principles (Lanjutan)

Accessibility
  • Color contrast
  • Keyboard navigation
  • Screen reader support
  • Alternative text
Responsiveness
  • Mobile-first design
  • Flexible layouts
  • Adaptive content
  • Touch-friendly interfaces

UI Design Elements

Visual Elements

  • Typography: Fonts, sizes, weights
  • Colors: Color palette, contrast
  • Images: Photos, icons, illustrations
  • Shapes: Buttons, cards, containers

Interactive Elements

  • Buttons: Primary, secondary, tertiary
  • Forms: Input fields, labels, validation
  • Navigation: Menus, breadcrumbs, pagination
  • Feedback: Loading states, error messages

Wireframing

Low-Fidelity
  • Basic layout
  • Content structure
  • Navigation flow
  • Quick iteration
Mid-Fidelity
  • Detailed layout
  • Content hierarchy
  • Interactive elements
  • User flow
High-Fidelity
  • Pixel-perfect design
  • Final content
  • Interactive prototype
  • Development ready

Prototyping

Prototype Types

  • Static Prototypes: Non-interactive mockups
  • Interactive Prototypes: Clickable interfaces
  • Animated Prototypes: Motion and transitions
  • Functional Prototypes: Working features

Prototyping Tools

  • Design Tools: Figma, Sketch, Adobe XD
  • Prototyping Tools: InVision, Marvel, Principle
  • Code Tools: HTML/CSS, React, Vue
  • Testing Tools: UserTesting, Maze, Lookback

Usability Testing

Moderated Testing
  • Facilitator present
  • Real-time feedback
  • Detailed insights
  • Higher cost
Unmoderated Testing
  • Self-guided
  • Remote testing
  • Scalable
  • Lower cost

Testing Methods

  • Task-Based Testing: Specific task completion
  • Think-Aloud Protocol: Verbal feedback
  • Eye Tracking: Visual attention analysis
  • A/B Testing: Comparative testing

Testing Metrics

  • Task Success Rate: Percentage of completed tasks
  • Time on Task: Time to complete tasks
  • Error Rate: Number of errors made
  • User Satisfaction: User feedback scores

UI Specifications

Visual Specifications
  • Color codes
  • Typography
  • Spacing
  • Dimensions
Interactive Specifications
  • Hover states
  • Click states
  • Transitions
  • Animations
Responsive Specifications
  • Breakpoints
  • Grid systems
  • Layout variations
  • Content scaling

Latihan Praktis

Tugas: Lakukan UX research untuk sistem perpustakaan

Langkah:
  1. Identifikasi target users
  2. Buat user personas
  3. Map user journey
  4. Identifikasi pain points
  5. Buat research report

Exercise 2: UI Design

Tugas: Desain UI untuk sistem perpustakaan

Langkah:
  1. Buat wireframes
  2. Desain visual interface
  3. Buat interactive prototype
  4. Test dengan users
  5. Iterasi berdasarkan feedback

Exercise 3: Usability Testing

Tugas: Lakukan usability testing

Langkah:
  1. Buat test plan
  2. Recruit test participants
  3. Conduct usability tests
  4. Analyze results
  5. Buat test report

Exercise 4: UI Specifications

Tugas: Buat UI specifications lengkap

Langkah:
  1. Dokumentasikan visual specs
  2. Dokumentasikan interactive specs
  3. Dokumentasikan responsive specs
  4. Buat style guide
  5. Validasi dengan development team

Proyek Integrasi: User Interface

Analisis UI untuk Sistem Perpustakaan Digital:

1. UX Research

User Personas

  • Primary Persona - Student: Active library users, tech-savvy, mobile-first
  • Secondary Persona - Librarian: Library staff, desktop users, efficiency-focused
  • Secondary Persona - Faculty: Research-focused, desktop users, advanced features

User Journey

Student Borrowing Journey

  • Discovery: Search for books
  • Evaluation: Check book details and availability
  • Action: Borrow or reserve book
  • Management: Track borrowing history
  • Return: Return books on time

Pain Points

  • Search Issues: Difficult to find books
  • Availability Issues: Unclear book availability
  • Mobile Experience: Poor mobile interface
  • Notification Issues: No real-time notifications

2. UI Design

Design System

  • Color Palette: Blue primary, green secondary, gray neutrals
  • Typography: Inter font family, 16px base size
  • Spacing: 8px grid system
  • Components: Consistent button, form, and card styles

Layout Design

  • Header: Logo, navigation, user menu
  • Sidebar: Quick access, filters, categories
  • Main Content: Book listings, search results
  • Footer: Links, contact information

Responsive Design

  • Mobile (320px-768px): Single column, touch-friendly
  • Tablet (768px-1024px): Two column, hybrid navigation
  • Desktop (1024px+): Multi-column, full navigation

3. Wireframes

Key Pages

  • Homepage: Search, featured books, quick access
  • Search Results: Book listings, filters, pagination
  • Book Details: Book information, availability, actions
  • User Dashboard: Borrowing history, reservations, fines

Prototypes

Interactive Prototypes

  • Low-Fidelity: Basic layout and navigation
  • Mid-Fidelity: Detailed interactions
  • High-Fidelity: Final design with animations

4. Usability Testing

Test Scenarios

  • Scenario 1: Search and borrow a book
  • Scenario 2: Reserve an unavailable book
  • Scenario 3: Check borrowing history
  • Scenario 4: Pay fines online

Test Results

Key Findings

  • Task Success Rate: 85%
  • Average Time: 2.5 minutes per task
  • Error Rate: 15%
  • User Satisfaction: 4.2/5

Issues Identified

  • Search Interface: Confusing search filters
  • Mobile Navigation: Difficult to access menu
  • Book Details: Missing key information
  • Borrowing Process: Too many steps

5. UI Specifications

Visual Specifications

  • Colors: Primary #3498db, Secondary #27ae60, Error #e74c3c
  • Typography: Inter Regular 16px, Bold 18px
  • Spacing: 8px, 16px, 24px, 32px
  • Components: Button height 44px, Input height 40px

Interactive Specifications

  • Hover States: 0.2s transition, color change
  • Click States: 0.1s transition, scale effect
  • Loading States: Skeleton screens, progress bars
  • Error States: Red borders, error messages

Responsive Specifications

  • Breakpoints: 320px, 768px, 1024px, 1440px
  • Grid System: 12-column grid, 20px gutters
  • Typography Scale: 14px mobile, 16px tablet, 18px desktop
  • Component Scaling: Responsive buttons, flexible layouts

Style Guide

Component Library

  • Buttons: Primary, secondary, outline variants
  • Forms: Input fields, labels, validation
  • Cards: Book cards, user cards, info cards
  • Navigation: Header, sidebar, breadcrumbs

Accessibility

  • Color Contrast: WCAG AA compliance
  • Keyboard Navigation: Tab order, focus states
  • Screen Reader: ARIA labels, alt text
  • Touch Targets: Minimum 44px touch areas
